Redirect Types

In TranslateProxy there are two modes of redirects, JS and HTTP (Simulated). We will be going over both of those and the pros and cons of both options

JS Redirect

JS Redirects behaves as if you directly typed the link into your URL box. The origin domain (ie: gtp.frankiee.me) will still be in the current session, this means if you were to press your browsers "Back" button it will eventually reach the origin domain. This mode is the most convenient, allowing you to quickly return to the origin website if you mess up the URL link for example.


Pro(s) Con(s)
Convenient Not Stealthy

HTTP (Simulated) Redirect

HTTP Redirects are when a domain is pointing to another domain/IP Address in their DNS Settings. The benefit of this is that because you are directly going to another IP Address so the origin domain is not stored in the session. Of course we can't always be changing where this domain is pointing to, so instead we do a simulated HTTP Redirect, it behaves very similar to a regular HTTP Redirect where the origin domain is not stored in the session history. This means if you press your browsers "Back" button, it will never reach the origin domain (Remember, it will still be in your browser's history log)


Pro(s) Con(s)
Stealthy Not Convenient
Kept in Browser's History Log (same with JS Redirects)
